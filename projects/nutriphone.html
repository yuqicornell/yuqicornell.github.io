NutriPhone

Sensemaking and confusion in designing interfaces for self-administered nutrient test result

Time: September 2014 - August 2015

Team Size: 4, advised by Dr. Lindsay Reynolds and Dr. Geri Gay

Role: UX Researcher/Designer. For this project, my job was focused on design research, usability study, and user testing

Methodology: User Testing, User Interview, Storyboarding, High-Fidelity Prototyping, Affinity Diagram, Secondary Research, User Survey

Product: Academic paper submitted to ACM CHI 2016 conference, interface prototype for Nutriphone mobile application

Introduction

Nutriphone is a smartphone based micronutrient testing system that utilizes phone cameras to analyze blood results for Vitamin B12, D, cholesterol, and other health markers.

We are exploring how individuals interpret different representations of their health data, cultural factors impacting use of the system, and the ways in which values and trust affect use of the system.

This project is a collaboration between the Interaction Design Lab, Cornell Mechanical Engineering and Nutritional Sciences, researchers at the University of Maryland, and industrial partners.

Online Survey Study – Comparing Feedback Interfaces

Five variations of interface design (number, color, text, graph, hydrid) were tested using with 249 MTurkers (Amazon’s Mechanical Turk). Each participant completed an online questionnaire of evaluating different result interface designs. Participants were asked to rank three designs they viewed in order of preference.

We used descriptive statistics and Fisher’s exact test for quantitative data analysis.

Lab Study - User Testing with Interactive Prototypes

Procedure

We conducted user studies with participants, using an interactive prototype of the NutriPhone system (N=27). After user studies, we also conducted a semi-structured interviews.

Participants were first asked to use an interactive prototype of NutriPhone application on their personal smartphone. There are two conditions: One condition displayed B12 levels is within recommended range, and the other is lower than the recommended range.

Following the user testing, we followed up with questions about how they understood (or did not understand) their test result, how they typically make sense of medical test results, and their overall impressions.

Data Analysis

We first transcribe all the interview recordings. Based on the transcriptions, three authors reviewed each interview and independently took notes about key ideas on sticky notes. We then used affinity diagrams to organize these ideas into the themes.
